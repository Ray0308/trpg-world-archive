# シナリオ登録フォーム — セットアップ手順

## プロジェクトの名前（呼び方）

| 名前 | 実体 | やること |
|------|------|----------|
| **DBPJ** | スプレッドシート（TRPG World Archive DB） | データが溜まる |
| **NPCPJ** | NPC用フォーム + `docs/gas-npc-form.gs` | サイト API・デプロイ（シナリオ読取もここ） |
| **シナリオPJ** | シナリオ入力フォーム + `docs/gas-scenario-form-bound.gs` | **シナリオ送信 → SCENARIOS 転記** |

> KP は **ID を知らなくてよい** 設計です。  
> 登場 NPC・組織・PC・関連シナリオは **すべて名前で入力** し、GAS が ID を自動解決します。

```
シナリオPJ（フォーム）─送信→ シナリオPJ（GAS）─書込→ DBPJ（SCENARIOS）
NPCPJ（GAS）────────────────読取────────→ サイト API（?type=scenarios）
```

## 手順 1 — Google フォームを作る

[Google フォーム](https://docs.google.com/forms) で新規フォームを作成します。

### フォーム全体の設定

| 項目 | コピペ用 |
|------|---------|
| **タイトル** | `YOKOFOLIA シナリオ登録` |
| **フォームの説明** | 下記「フォーム説明文」をそのまま貼り付け |

#### フォーム説明文（コピペ用）

```
YOKOFOLIA ふわっと住民台帳にシナリオを登録するフォームです。

【ID は不要です】
登場 NPC・組織・PC・関連シナリオは、サイトに表示されている名前をそのまま書いてください。
カンマ区切りで複数入力できます。

【入力のコツ】
・必須は「シナリオ名」と「概要」だけです
・わからない項目は空欄でOKです
・送信後も編集リンクから内容を直せます
```

### 質問項目（この順・この文言で）

| # | 質問タイプ | 質問文 | 必須 | 説明文（コピペ用） |
|---|-----------|--------|------|-------------------|
| 1 | 記述式（短い回答） | シナリオ名 | ✅ | サイトに表示される正式名称です。 |
| 2 | 記述式（短い回答） | 年代 | | 例: `1920年代` `現代` |
| 3 | 記述式（段落） | 概要 | ✅ | シナリオカードに載る説明です。 |
| 4 | 記述式（短い回答） | 登場NPC | | NPC 名をカンマ区切りで入力（空欄可）。<br>例: `H.P. ラヴクラフト, ロバート・E・ハワード`<br>※ **NPC が先に登録されている必要があります** |
| 5 | 記述式（短い回答） | 登場組織 | | 組織名をカンマ区切りで入力（空欄可）。<br>例: `ミスカトニック大学` |
| 6 | 記述式（短い回答） | 関連PC | | PC 名をカンマ区切りで入力（空欄可）。<br>※ PCS シートに登録済みの名前と一致させてください |
| 7 | 記述式（短い回答） | 関連シナリオ | | 関連するシナリオ名をカンマ区切りで入力（空欄可）。<br>例: `黄衣の王の招き, 失われた都市` |
| 8 | 記述式（段落） | 備考 | | KP 用のメモです。PL サイトには表示されません。 |

> Google フォームの説明欄では `<br>` は使えないので、**改行は Enter で入れてください**。

### 質問文の注意

GAS は質問文の **「シナリオ名」「登場NPC」…** をそのまま読み取ります。  
`1. シナリオ名` のように番号を付けても自動で除去されますが、**文言は上表と同じ**にしてください。

**ID 用の質問は作らないでください**（KP には見えません）。

## 手順 2 — 回答先スプレッドシート

**NPC フォームと同じスプレッドシート** を回答の保存先に設定します。

1. フォーム編集画面 → **回答** タブ
2. 「スプレッドシートにリンク」→ **既存のスプレッドシートを選択**（NPC 用 DBPJ）
3. 新しい回答シートができる → そのままでOK（GAS が `SCENARIOS` シートに書き込みます）

---

## 手順 3 — シナリオPJ（GAS）

1. **シナリオ入力フォーム** → ⋮ → **スクリプトエディタ**（シナリオPJ）を開く
2. `docs/gas-scenario-form-bound.gs` の内容を `Code.gs` に貼り付けて保存
3. **トリガー** を追加:

| 項目 | 値 |
|------|-----|
| 実行する関数 | `onScenarioFormSubmit` |
| イベントのソース | **フォームから** |
| イベントの種類 | **フォーム送信時** |

4. （任意）エディタで `importAllScenarioResponses` を ▶ 実行 — 過去の回答を一括転記
5. （任意）`checkScenarioSetup` を ▶ 実行 — シート列の確認

---

## 手順 4 — NPCPJ の再デプロイ

シナリオを PL / KP サイトで表示するには、**NPCPJ** にシナリオ API が必要です。

1. NPC フォーム用 GAS プロジェクトを開く
2. `docs/gas-npc-form.gs` を最新版に更新して保存
3. **デプロイ** → **新しいデプロイ** → ウェブアプリ
4. デプロイ後、`?type=version` で `scenarios` / `scenario-visibility` が capabilities に含まれることを確認

---

## 手順 5 — テストとリンク設定

1. シナリオフォームを1件送信
2. DBPJ の `SCENARIOS` シートに行が追加されることを確認
   - `id` は `scn_001` のように自動採番
   - `npc_ids` などは名前から自動解決
3. フォームの **送信後の URL** をコピー
4. `js/links.js` の `scenarioForm` に貼り付け
5. GitHub に push → KP ページの「シナリオ登録フォーム」が **準備中** ではなくなる

---

## SCENARIOS シート列（参考）

| 列 | 内容 |
|----|------|
| id | 自動採番 `scn_###` |
| title | シナリオ名 |
| era | 年代 |
| summary | 概要 |
| npc_names / npc_ids | 登場 NPC（名前入力 → ID 自動） |
| organization_names / organization_ids | 登場組織 |
| pc_names / pc_ids | 関連 PC |
| related_scenario_names / related_scenario_ids | 関連シナリオ |
| memo | 備考（KP のみ） |
| pl_hidden | PL 非表示フラグ |
| edit_url | フォーム編集 URL |

---

## トラブルシュート

| 症状 | 対処 |
|------|------|
| フォーム送信しても `SCENARIOS` に行がない | シナリオPJ のトリガー `onScenarioFormSubmit` を確認 |
| KP の「表示設定」で `unknown type` | NPCPJ を再デプロイ（古い API） |
| 名前を入れたのにリンクされない | NPC / 組織が先に登録されているか、表記がサイト表示名と一致しているか確認 |
| PL にシナリオが出ない | `pl_hidden` がオンになっていないか、ハードリフレッシュ |

# 組織登録フォーム — セットアップ手順

## プロジェクトの名前（呼び方）

| 名前 | 実体 | やること |
|------|------|----------|
| **DBPJ** | スプレッドシート（TRPG World Archive DB） | データが溜まる（手でシートを作らない） |
| **NPCPJ** | NPC用フォーム + `docs/gas-npc-form.gs` | NPC登録・サイトAPI・デプロイ |
| **組織PJ** | 組織入力フォーム + `docs/gas-org-form-bound.gs` | **組織送信 → ORGANIZATIONS 転記** |

> 組織フォームを送っても **NPCPJ のスクリプトは動きません**（別フォームだから）。  
> 組織の転記は **組織PJ** に GAS を置きます。

```
組織PJ（フォーム）─送信→ 組織PJ（GAS）─書込→ DBPJ（ORGANIZATIONS）
NPCPJ（フォーム）─送信→ NPCPJ（GAS）─書込→ DBPJ（NPCS）
NPCPJ（GAS）────────────────読取────────→ サイト API
```

組織の名称・概要などを Google フォームで登録し、**NPC と同じ DBPJ** の `ORGANIZATIONS` シートに保存します。

> **NPC との紐づけは組織フォームでは不要です。**  
> 「この NPC はどの組織所属か」は **NPC 登録フォーム** の「所属組織」で設定します（組織名でOK。`organization_ids` は GAS が自動解決）。  
> 組織ページのメンバー一覧は、サイトが名前・ID の両方から自動で集計します。

## 手順 1 — Google フォームを作る

[Google フォーム](https://docs.google.com/forms) で新規フォームを作成します。

### フォーム全体の設定

| 項目 | コピペ用 |
|------|---------|
| **タイトル** | `YOKOFOLIA 組織登録` |
| **フォームの説明** | 下記「フォーム説明文」をそのまま貼り付け |

#### フォーム説明文（コピペ用）

```
YOKOFOLIA ふわっと住民台帳に組織を登録するフォームです。

【メンバー（NPC）について】
所属 NPC の登録はここでは行いません。
NPC 登録フォームの「所属組織」から紐づけてください。
組織ページのメンバー一覧は、サイトが自動で表示します。

【入力のコツ】
・必須は「組織名」と「概要」だけです
・わからない項目は空欄でOKです
・送信後も編集リンクから内容を直せます
```

### 質問項目（この順・この文言で）

各質問の **「説明」** 欄（質問文の下に表示されるヘルプ）に、表の「説明文」をコピペしてください。

| # | 質問タイプ | 質問文 | 必須 | 説明文（コピペ用） |
|---|-----------|--------|------|-------------------|
| 1 | 記述式（短い回答） | 組織名 | ✅ | サイトに表示される正式名称です。<br>例: `ミスカトニック大学` |
| 2 | **ファイルアップロード** または 記述式（短い回答） | アイコン | | **画像**: ファイルを1つアップロード（推奨）<br>**または絵文字**: `🏛️` `🔬` など1つ<br>空欄の場合は 🏛️ |
| 3 | 記述式（短い回答） | 概要 | ✅ | 組織カードに載る1〜2行の短い説明です。<br>例: `アーカムに存在する大学。考古学・民俗学など多様な学部を擁する。` |
| 4 | 記述式（段落） | 説明 | | 詳しい説明です。改行して複数段落を書いても大丈夫です。<br>例:<br>`1880年に設立された私立大学。キャンパス内には考古学博物館がある。`<br>`表向きは普通の総合大学だが、禁断の知識に触れる研究者も少なくない。` |
| 5 | 記述式（短い回答） | 所在地 | | 地名や場所を自由に書いてください。<br>例: `アーカム` `マサチューセッツ州アーカム` |
| 6 | 記述式（短い回答） | 関連シナリオ | | 関連するシナリオ ID をカンマ区切りで入力（空欄可）。<br>例: `scn-lost-city, scn-yellow-invite`<br>※ ID がわからなければ空欄でOKです |
| 7 | 記述式（段落） | 備考 | | KP 用のメモです。PL サイトには表示されません。<br>例: `シナリオ第3章で再登場予定` |

> Google フォームの説明欄では `<br>` は使えないので、**改行は Enter で入れてください**。上表の `<br>` は段落の区切りの目安です。

### 質問文の注意

GAS は質問文の **「組織名」「概要」…** をそのまま読み取ります。  
`1. 組織名` のように番号を付けても自動で除去されますが、**文言は上表と同じ**にしてください。

## 手順 2 — 回答先スプレッドシート

**NPC フォームと同じスプレッドシート** を回答の保存先に設定します。

1. フォーム編集画面 → **回答** タブ
2. 「スプレッドシートにリンク」→ **既存のスプレッドシートを選択**（NPC 用）
3. 新しいシート `フォームの回答 2` ができる → そのままでOK（データは GAS が `ORGANIZATIONS` シートに書き込みます）

---

## 手順 3 — GAS を更新

1. [Google Apps Script](https://script.google.com/) で **NPC 用プロジェクト** を開く
2. `docs/gas-npc-form.gs` の内容を **すべて** `Code.gs` に貼り付けて保存  
   （`onOrganizationFormSubmit` / `getPublicOrganizations_` が含まれていること）
3. **デプロイ → デプロイを管理 → 編集 → 新バージョン → デプロイ**

### 動作確認

```
.../exec?type=version
→ api_version に org-form が含まれる

.../exec?type=organizations
→ [] または組織の配列
```

---

## 手順 4 — 組織PJ に GAS + トリガー

1. **組織入力フォーム** → ⋮ → **スクリプトエディタ**（**組織PJ**）を開く
2. `docs/gas-org-form-bound.gs` を **すべて** `Code.gs` に貼り付けて **保存**
3. 左 **トリガー** → **トリガーを追加**:

| 項目 | 値 |
|------|-----|
| 実行する関数 | `onOrganizationFormSubmit` |
| イベントのソース | **フォームから** |
| イベントの種類 | **フォーム送信時** |

4. **保存**（権限承認）
5. （任意）関数 **`importAllOrgResponses`** を ▶ 実行 → 溜まった回答を一括取り込み

**デプロイは不要**（組織PJ は転記だけ。API は NPCPJ のデプロイ）。

### NPCPJ 側で消してよいもの

- NPCPJ の `onOrganizationFormSubmit` トリガー（あれば）
- NPCPJ の `onSpreadsheetFormSubmit` トリガー（あれば）

### NPCPJ で必要なこと（API用）

`docs/gas-npc-form.gs` を貼って **ウェブアプリを再デプロイ**（`?type=organizations` 用）。  
組織の**転記**は組織PJ、**公開**は NPCPJ です。

---

## 手順 5 — テスト & サイト設定

1. 組織フォームを1件送信
2. スプレッドシートに **`ORGANIZATIONS`** シートができ、`org_001` などの行が入ることを確認
3. フォームの **送信後の URL** をコピー
4. `js/links.js` の `organizationForm` に貼り付け:

```javascript
organizationForm: 'https://docs.google.com/forms/d/e/xxxxxxxx/viewform',
```

5. GitHub に push → KP ページの「組織登録フォーム」が **準備中** ではなくなる

---

## ORGANIZATIONS シートの列

初回送信時に自動作成されます。

| 列 | 内容 |
|----|------|
| id | 自動採番 `org_001`, `org_002` … |
| name | 組織名 |
| icon | アイコン |
| summary | 概要 |
| description | 説明 |
| location_id | （将来用・フォームでは空） |
| location_name | 所在地 |
| scenario_ids | 関連シナリオ ID |
| memo | 備考（API 非公開） |
| pl_hidden | PL 非表示（KP から設定・第二弾） |
| edit_url | フォーム編集 URL |
| created_at / updated_at | 登録・更新日時 |

手動でシートを作る場合は `resetOrganizationSheetForProduction` を GAS から1回実行してもOKです。

## うまくいかないとき

| 確認 | 意味 |
|------|------|
| スプシに **`フォームの回答 2`**（など）に行がある | フォーム→スプシはOK。GAS が動いていないだけ |
| `checkOrgSetup` で「回答シート: 見つかりません」 | 質問文が **組織名** になっているか、回答先スプシが合っているか確認 |
| `importLastOrgResponse` で取り込める | コードはOK。**トリガーだけ**未設定 |
| 入力PJ / 組織フォーム側のトリガー | 使わず **スプレッドシート側**（手順4）に切り替え |

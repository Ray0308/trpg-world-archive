# 組織登録フォーム — セットアップ手順

組織の名称・概要・説明などを Google フォームで登録し、**NPC と同じスプレッドシート** の `ORGANIZATIONS` シートに保存します。

> **NPC との紐づけは組織フォームでは不要です。**  
> 「この NPC はどの組織所属か」は **NPC 登録フォーム** の「所属組織」で設定します。組織ページのメンバー表示はサイトが自動で集計します。

## 手順 1 — Google フォームを作る

[Google フォーム](https://docs.google.com/forms) で新規フォームを作成し、タイトルを **「YOKOFOLIA 組織登録」** などにします。

### 質問項目（この順・この文言で）

| # | 質問タイプ | 質問文 | 必須 |
|---|-----------|--------|------|
| 1 | 記述式（短い回答） | 組織名 | ✅ |
| 2 | 記述式（短い回答） | 英語名 | |
| 3 | 記述式（短い回答） | アイコン | |
| 4 | 記述式（短い回答） | 概要 | ✅ |
| 5 | 記述式（段落） | 説明 | |
| 6 | 記述式（短い回答） | 所在地 | |
| 7 | 記述式（短い回答） | 関連シナリオ | |
| 8 | 記述式（段落） | 備考 | |

### 入力のヒント（フォームの説明欄に書いておくとよい）

- **アイコン** … 絵文字1つ（例: `🏛️`）。空欄なら 🏛️ になります。
- **概要** … 1〜2行の短い説明（組織カードに表示）
- **説明** … 詳しい説明（改行OK。段落はそのまま保存されます）
- **所在地** … 地名の自由記述（例: アーカム）。`location_id` は第二弾で対応予定
- **関連シナリオ** … シナリオ ID をカンマ区切り（例: `scn-lost-city`）。空欄でもOK
- **備考** … KP 用メモ（PL サイトには出ません）

### 質問文の注意

GAS は質問文の **「組織名」「概要」…** をそのまま読み取ります。  
`1. 組織名` のように番号を付けても自動で除去されますが、**文言は上表と同じ**にしてください。

## 手順 2 — 回答先スプレッドシート

**NPC フォームと同じスプレッドシート** を回答の保存先に設定します。

1. フォーム編集画面 → **回答** タブ
2. 「スプレッドシートにリンク」→ **既存のスプレッドシートを選択**（NPC 用）
3. 新しいシート `フォームの回答 2` ができる → そのままでOK（データは GAS が `ORGANIZATIONS` シートに書き込みます）

---

## 手順 3 — GAS を更新

1. [Google Apps Script](https://script.google.com/) で **NPC 用プロジェクト** を開く
2. `docs/gas-npc-form.gs` の内容を **すべて** `Code.gs` に貼り付けて保存  
   （`onOrganizationFormSubmit` / `getPublicOrganizations_` が含まれていること）
3. **デプロイ → デプロイを管理 → 編集 → 新バージョン → デプロイ**

### 動作確認

```
.../exec?type=version
→ api_version に org-form が含まれる

.../exec?type=organizations
→ [] または組織の配列
```

---

## 手順 4 — 送信トリガー

1. GAS エディタ → **トリガー**（時計アイコン）→ **トリガーを追加**
2. 設定:

| 項目 | 値 |
|------|-----|
| 実行する関数 | `onOrganizationFormSubmit` |
| イベントのソース | フォームから |
| フォーム | **YOKOFOLIA 組織登録**（手順1で作ったフォーム） |
| イベントの種類 | フォーム送信時 |

3. 保存（初回は権限の承認が必要）

---

## 手順 5 — テスト & サイト設定

1. 組織フォームを1件送信
2. スプレッドシートに **`ORGANIZATIONS`** シートができ、`org_001` などの行が入ることを確認
3. フォームの **送信後の URL** をコピー
4. `js/links.js` の `organizationForm` に貼り付け:

```javascript
organizationForm: 'https://docs.google.com/forms/d/e/xxxxxxxx/viewform',
```

5. GitHub に push → KP ページの「組織登録フォーム」が **準備中** ではなくなる

---

## ORGANIZATIONS シートの列

初回送信時に自動作成されます。

| 列 | 内容 |
|----|------|
| id | 自動採番 `org_001`, `org_002` … |
| name | 組織名 |
| name_en | 英語名 |
| icon | アイコン |
| summary | 概要 |
| description | 説明 |
| location_id | （将来用・フォームでは空） |
| location_name | 所在地 |
| scenario_ids | 関連シナリオ ID |
| memo | 備考（API 非公開） |
| pl_hidden | PL 非表示（KP から設定・第二弾） |
| edit_url | フォーム編集 URL |
| created_at / updated_at | 登録・更新日時 |

手動でシートを作る場合は `resetOrganizationSheetForProduction` を GAS から1回実行してもOKです。
